# Логика расчёта цены окна

## Принцип
Калькулятор «обучается» на реальных ценах из производственной программы.
Чем больше точек цен ты вводишь — тем точнее расчёт.

## Что влияет на цену (23 параметра)
Всё, **кроме направления открывания створки** (лево/право).

| # | Параметр | Тип | Как влияет |
|---|----------|-----|-----------|
| 1 | Площадь (ш × в) | число м² | Основной фактор |
| 2-6 | Профиль | выбор одного | Разные бренды = разная цена |
| 7-10 | Стеклопакет | выбор одного | Двухкамерный > однокамерного, энергосбер. дороже |
| 11 | Поворотные створки | кол-во | Каждая добавляет к цене |
| 12 | Поворотно-откидные створки | кол-во | Дороже поворотных |
| 13 | Фрамуги | кол-во | Добавляют к цене |
| 14-17 | Фурнитура (Roto/Vorne/Verta/Futuruss) | кол-во по типу | Roto дороже Vorne и т.д. |
| 18 | Ламинация внутренняя | есть/нет | Цветная = дороже |
| 19 | Ламинация наружная | есть/нет | Цветная = дороже |
| 20 | Сэндвич-панели | кол-во | Дешевле стеклопакета |
| 21 | Импосты | кол-во | Каждый добавляет |
| 22 | Москитные сетки | кол-во | Каждая добавляет |
| 23 | Цветные ручки | кол-во | Цветная дороже белой |

## Алгоритм (Ridge Regression)

### При 1 точке цены
Простая пропорция по площади:
```
цена = точка.цена × (площадь / точка.площадь)
```

### При 2+ точках
Линейная регрессия с регуляризацией:
```
цена = w0 + w1×площадь + w2×Brusbox + w3×KBE + ... + w23×цветная_ручка
```

Веса (w0...w23) вычисляются автоматически методом Ridge Regression:
- Строится матрица фич X из всех обученных точек
- Решается: w = (X^T × X + λI)^(-1) × X^T × y
- λ = 10 (предотвращает переобучение при малом кол-ве точек)

### Как улучшить точность
1. Вводи цены на **разные** конфигурации (не только на одинаковые окна разного размера)
2. Меняй **один параметр за раз** — так система точнее поймёт сколько стоит каждый компонент
3. Минимум 5-10 точек для базовой точности, 20+ для хорошей

### Пример обучения
1. Глухое 1000×1000, Rehau 60, однокамерный → ввёл 5000 р.
2. Глухое 1400×1400, Rehau 60, однокамерный → ввёл 8000 р. (система поняла: площадь влияет)
3. Глухое 1000×1000, Rehau 60, двухкамерный → ввёл 5800 р. (система поняла: двухкамерный +800 р.)
4. 1 створка п/о 1000×1000, Rehau 60, однокамерный → ввёл 7500 р. (система поняла: створка п/о +2500 р.)

## Хранение данных
- **localStorage** ключ `okna_prices` — массив JSON с точками цен
- Каждая точка хранит: id, профиль, стеклопакет, размеры, площадь, вектор фич (23 числа), цена, дата
- Ключ `okna_settings` — скидка и наценка

## Формула для клиента
```
Цена производства = оценка по регрессии
Цена со скидкой = Цена производства × (100 - скидка%) / 100
Цена клиента = Цена со скидкой × (100 + наценка%) / 100
  или: Цена со скидкой + наценка в рублях
```
